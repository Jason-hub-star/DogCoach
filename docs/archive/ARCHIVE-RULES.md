# Archive Rules (TailLogweb)

`docs/archive`는 운영 중인 의사결정/상태 문서와 분리해, 완료된 기록을 보관하는 영역입니다.

## 이동 기준

아래 조건을 모두 만족하면 archive 이동 대상입니다.

1. 현재 운영 판단에 직접 사용되지 않는다.
2. 동일 사실이 `docs/status/*` 또는 `docs/ref/*` 정본에 반영되어 있다.
3. 재발/감사 추적을 위해 기록 보존은 필요하다.
4. 파일 이동 후에도 링크 추적이 가능하다(원본 위치를 로그에 남김).

## 우선 이동 대상

- `DECISION-LOG`에서 해결된 결정 항목
- 특정 날짜의 일회성 실행 보고서(현재 판단에 직접 쓰이지 않는 경우)
- 완료된 임시 점검/실험 문서

## 이동 금지 대상

- 현재 `in progress` 상태를 설명하는 문서
- 배포/운영 runbook 정본
- 스키마/아키텍처 정본(`docs/ref/*`)

## 실행 절차

1. 이동 전: `docs/status/PROJECT-STATUS.md`에서 해당 항목 상태를 `done (YYYY-MM-DD)`로 확정
2. 이동: 파일을 `docs/archive/...`로 이동
3. 링크: 원본 인덱스 문서에 archive 경로를 추가
4. 검증: 깨진 링크/중복 사실 여부 확인
